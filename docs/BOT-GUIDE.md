## 1. ภาพรวมการทำงาน

```
ข้อความ LINE
    │
    ├─ พิมพ์ "ช่วย" ──────────────► คู่มือคำสั่ง
    │
    ├─ แก้ / ปรับ / ลบ ──────────► Correction (regex → Gemini สำรอง)
    │                                 └─ บันทึก + ตอบสรุป
    │
    └─ ข้อมูลการเงิน ─────────────► Gemini parse
                                      └─ merge regex (หมู/ได้/จ่าย)
                                      └─ carry-forward จากวันก่อน
                                      └─ บันทึก + ตอบสรุป
```

Bot รองรับ **ภาษาพูดไทย** — ไม่ต้องพิมพ์ตามฟอร์มเป๊ะๆ แต่ยิ่งชัดยิ่งแม่น

## 2. บันทึกยอดประจำวัน

ส่งข้อความที่มีตัวเลข + คำที่ bot รู้จัก เช่น โอน, สด, หมู, ได้, จ่าย

### รายรับ

| รูปแบบ | ความหมาย |
|--------|----------|
| `โอน 5000` | รายรับโอนเงิน |
| `สด 3000` / `เงินสด 3000` | รายรับเงินสด |
| `delivery 800` | รายรับ Delivery |
| `ได้คนละครึ่ง 1265` | รายรับพิเศษ (โปรคนละครึ่ง) |
| `ได้ไลน์แมน 450` | รายรับพิเศษ (Lineman) |
| `รับ...` / `รายรับ...` | รายรับพิเศษ |

> **กฎ:** บรรทัดที่ขึ้นต้นด้วย **ได้ / รับ / รายรับ / รายได้** = รายรับเสมอ (ไม่ไปอยู่รายจ่าย)

### รายจ่าย

| รูปแบบ | ความหมาย |
|--------|----------|
| `จ่ายแม็คโคร 3800` | รายจ่ายทั่วไป |
| `ซื้อของ แม็คโคร 1300` | รายจ่ายทั่วไป |
| `ค่าแรง 1500` | ค่าแรง |
| `แก๊ส 150` | ค่าแก๊ส |
| `น้ำแข็ง 35` | ค่าน้ำแข็ง |

> **กฎ:** บรรทัดที่ขึ้นต้นด้วย **จ่าย / ซื้อ** = รายจ่ายเสมอ

### ตัวอย่างบันทึกยอดเต็มวัน

```
หนองปิง
โอน 5000 สด 3000
แดง4 สับ3 มันหมู 8
ได้คนละครึ่ง 1265
จ่ายแม็คโคร 3800
```

Bot จะตอบสรุปรายรับ / รายจ่าย / กำไร และแจ้งค่าที่ดึงจากวันก่อน (ถ้ามี)

## 3. คำสั่งแก้ไข / ปรับ / ลบ

ใช้เมื่อบันทึกไปแล้วแต่ต้องการแก้ — **ไม่ต้องส่งยอดทั้งวันใหม่**

พิมพ์ **`ช่วย`** ใน LINE เพื่อดูคำสั่งทั้งหมด

### แก้ยอดเงิน / ค่าใช้จ่าย

```
แก้ โอน 3000
ปรับค่าแรง 850
ค่า 850              ← ย่อ = ค่าแรง
แก้ ค่าเช่า 5000
แก้ ค่าไฟฟ้า 1200
```

### แก้ราคาหมู (ต่อกิโล)

```
ปรับหมูสับราคา 120
ปรับหมูแดง ราคา 130
ปรับมันหมูราคา 65
```

รองรับคำติดกัน — ระบบจะแยกให้อัตโนมัติ เช่น `ปรับหมูสับราคา 120` → `ปรับ หมูสับ ราคา 120`

เมื่อบันทึกราคาแล้ว แต่ยังไม่มีจำนวน Bot จะแสดง:

```
฿120/กก. (⏳ รอยอดจำนวน)
```

จากนั้นพิมพ์จำนวนได้เลย:

```
แดง4
สับ3
```

### ลบรายการ

```
ลบ แม็คโคร
ลบ โอน
```

## 4. คำศัพท์หมู

Bot รู้จักคำย่อและคำพ้องความหมาย:

| ในระบบ | คำที่รู้จัก |
|--------|------------|
| **หมูแดง** | `หมูแดง`, `หมูเนื้อ`, `แดง`, `แดง4` |
| **หมูสับ** | `หมูสับ`, `สับ`, `สับ3` |
| **มันหมู** | `มันหมู`, `หมูมัน`, `มัน`, `มันหมู 8` |

### รูปแบบการพิมพ์

```
หมูแดง 8 กก. × 130        ← จำนวน + ราคา
แดง4                       ← จำนวนอย่างเดียว (ใช้ราคาจากวันก่อน)
ปรับหมูสับราคา 120         ← ราคาอย่างเดียว
```

## 5. สาขา

ใส่ชื่อสาขา **บรรทัดแรก** ของข้อความ:

| คำที่พิมพ์ | สาขา |
|-----------|------|
| `ญี่ปุ่น` / `ตลาดญี่ปุ่น` | ก๋วยเตี๋ยวไทยครูตอมตลาดญี่ปุ่น |
| `หนองปิง` / `สายหนองปิง` | ก๋วยเตี๋ยวไทยครูตอมสายหนองปิง |

```
ญี่ปุ่น
โอน 3385 สด 1000
```

ถ้าไม่ระบุสาขา ใช้ค่า default จากระบบ (`DEFAULT_SHOP_ID`)

## 6. Carry-forward อัตโนมัติ

เมื่อบันทึกวันใหม่ แต่ไม่ได้พิมพ์ค่าบางอย่าง Bot จะ **ดึงจาก record ล่าสุดของสาขานั้น**:

| รายการ | พฤติกรรม |
|--------|----------|
| ราคาหมู (แดง/สับ/มัน) | ดึงราคา/กก. เมื่อพิมพ์แค่จำนวน เช่น `แดง4` |
| ค่าแรง, น้ำแข็ง, แก๊ส | ดึงจากวันก่อน |
| ค่าเช่า, ค่าน้ำ, ค่าไฟฟ้า | ใส่ในรายจ่ายพิเศษอัตโนมัติ |

Bot จะแจ้งในข้อความตอบกลับว่าดึงค่าอะไรจากวันก่อนบ้าง

## 9. สถาปัตยกรรม Hybrid Parser

### บันทึกยอด (`parseFinancialMessage`)

1. **Gemini** อ่านภาษาพูด → JSON
2. **`applyDeterministicEnrichment()`** รวมผล:
   - **ได้/จ่าย** → regex เป็นหลัก (Gemini มักผิด)
   - **หมู/โอน/สด/ค่าแรง** → เอาค่าที่มีข้อมูลจากทั้ง Gemini และ regex
3. **`getCarriedDefaults()`** ดึงค่าจากวันก่อน
4. **`upsertParsedRecord()`** บันทึก Supabase

### คำสั่งแก้ไข (`applyLineCorrection`)

1. **`normalizeCommandText()`** + **`normalizeNaturalCommandLine()`** แยกคำติดกัน
2. **`parseCorrectionMessage()`** — regex เป็นหลัก
3. ถ้า regex ได้ 0 actions แต่ข้อความมี `หมู|แดง|สับ|มัน|ปรับ|แก้` → **`parseCorrectionWithGemini()`**
4. **`buildCorrectionSummary()`** สรุปเป็นภาษาคน

### ไฟล์หลัก

| ไฟล์ | หน้าที่ |
|------|--------|
| `src/lib/services/financial-parser.service.ts` | Gemini parse, regex, hybrid merge, ข้อความตอบกลับ |
| `src/lib/services/financial-correction.service.ts` | คำสั่ง แก้/ปรับ/ลบ |
| `src/lib/services/smart-command.service.ts` | แยกคำติดกัน + สรุป correction |
| `src/lib/services/recurring-expenses.service.ts` | carry-forward ค่าใช้จ่ายประจำ |
| `src/lib/services/financial-records.service.ts` | บันทึก DB + correction flow |
| `src/lib/services/webhook-processor.service.ts` | จัดเส้นทางข้อความ LINE |

## 10. แก้ปัญหาเวลาใช้งาน

### Bot ตอบ "ไม่เข้าใจคำสั่งแก้ไข"

- พิมพ์ **`ช่วย`** ดูรูปแบบที่รองรับ
- ลองใส่ชื่อสาขาบรรทัดแรก เช่น `ญี่ปุ่น` แล้วค่อยคำสั่งแก้
- ตัวอย่างที่ใช้ได้: `ปรับ หมูสับ ราคา 120` (มีเว้นวรรค)

### Bot ไม่บันทึกเป็นข้อมูลการเงิน

ข้อความต้องมีอย่างน้อยหนึ่งอย่าง:

- ตัวเลข + `โอน` / `สด` / `delivery`
- คำเกี่ยวกับหมู: `แดง`, `สับ`, `มันหมู`
- `ได้...` / `จ่าย...` / `ซื้อ...`
- ชื่อสาขา + ตัวเลข

### ราคาหมูไม่ขึ้น

- ถ้าแก้แค่ราคา → ต้องเห็น `฿xxx/กก.` ในข้อความตอบกลับ
- พิมพ์จำนวนตาม: `แดง4` `สับ3`
- ตรวจใน Dashboard ว่าเลือกสาขาถูก

### Dashboard ข้อมูลเก่า

- เปิดจาก URL จริง: `https://line-dashboard-bot.vercel.app/dashboard.html`
- กดเชื่อมต่อ API Key ให้ครบ
- สลับแท็บกลับมา — ระบบ refresh อัตโนมัติ

### ตรวจสอบระบบ (สำหรับ admin)

```bash
curl https://line-dashboard-bot.vercel.app/api/health
```
